# CLAUDE.md — Stage1無料診断Webアプリ(リポジトリ直下に置く)

## プロジェクトの目的
「障害対応スキルマップ Stage1」の無料セルフチェック(Q1〜Q10)をWebアプリ化する。
役割は集客装置: 3分で診断 → Lv結果表示 → 結果をXでシェア → note/Xへ導線。

## 仕様の唯一の情報源
- `docs/03_診断アプリ_仕様書.md` … Q1〜Q10の設問・選択肢・正答、Lv判定ロジック、Lv別結果文。**この文言を改変しない**(判断基準DB v7から抽出した確定仕様)。
- 判定は高Lv優先(Lv10→Lv9→Lv8→…)。Lv8以上はQ1〜Q6全問正解必須、Q7はLv9以上で必須。Q1不正解はLv0(責めない文面)。
- 結果画面に簡易版注記(仕様書冒頭の注記文言)を必ず表示する。

## 技術方針
- 静的サイト(サーバ不要)。Vite + React + TypeScript、または素のHTML/JS単一ファイルでもよい。GitHub Pagesで公開。
- 回答・判定はすべてクライアントサイド。個人情報は収集しない。localStorageも使わない(状態はメモリのみ)。
- 計測はプライバシーに配慮した軽量アクセス解析(なくても可。まず公開を優先)。

## 画面フロー
1. トップ: 「あなたの障害対応、今どのレベル?」+ 所要3分 + 開始ボタン
2. 設問: 1問1画面、進捗バー(1/10)、選択肢の順序は毎回シャッフル(正答位置の偏り防止)
3. 結果: Lv名 + 冒頭文 + できていること + 次に身につけたいこと + おすすめの次の行動(仕様書の結果文をそのまま)
4. シェア: 「私は Lv◯ ◯◯者 でした」のXシェアボタン(結果詳細は含めず、Lvと診断URLのみ)
5. 導線: note固定記事へのリンク1つ + X(@PpkunSe)フォローボタン

## 実装の優先順位
1. 動く最小版を最短で公開(デザインは後回し)
2. シェア文言とOGP画像
3. Lv別の導線出し分け(低Lv→無料記事、高Lv→リーダー向けコンテンツ)

## テスト
- 判定ロジックはpytest相当(vitest)で全Lvの境界ケースを網羅する。
  例: Q1〜Q10全正解=Lv10 / Q1〜Q9正解=Lv9 / Q1〜Q6+Q8=Lv8 / Q1〜Q7=Lv7 / Q7のみ不正解でQ1〜Q6+Q8〜Q10正解=Lv8(Q7はLv9以上で必須のため) / Q1不正解=Lv0
- 文言は仕様書と一字一句比較するスナップショットテストを入れる。

## やらないこと
- 正式診断(111観察チェック・AI採点)の実装。これは別商品。
- ログイン、メールアドレス収集、DB。まず配布を優先する。
