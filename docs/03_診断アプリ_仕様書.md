## 設問(Q1〜Q10・4択・正答固定)

### Q1(対応Lv1: アラート受信者)
**質問**: アラート通知を受け取った直後、最初に何をしますか？
- A. 原因を調べるため、すぐにログを確認する
- B. 通知の重要度を見て、緊急そうなら担当者へ転送する
- C. 対応が必要な通知か確認し、記載された時刻・対象・事象をメモする
- D. 同じ通知がもう一度届くまで様子を見る
**正答**: C / 測定スキル: S001 / S003
判定意図: 対応要否を確認し、初動に必要な事実を残す意識を確認する

### Q2(対応Lv2: アラート読解者)
**質問**: 見慣れない機器名が記載されたアラートを受け取りました。最初の確認として適切なのはどれですか？
- A. エラー文をインターネットで検索し、原因候補を調べる
- B. 構成図や運用資料を確認し、対象機器が属するシステム・役割・検知された異常を整理する
- C. 機器名を知っていそうな人へ、そのままアラートを転送する
- D. 過去に見たことがないため、重要度は低いと判断する
**正答**: B / 測定スキル: S002 / S018 / S019 / S020
判定意図: 対象システム・機器役割・検知異常を資料から確認する意識を測る

### Q3(対応Lv3: 影響確認者)
**質問**: アラートを確認したところ、原因はまだ分かりません。次に優先して確認することはどれですか？
- A. 原因を特定するため、関連しそうなログをすべて確認する
- B. 手順書や運用資料を確認し、異常が続いているか、利用者や業務へ影響が出ているかを確認する
- C. 同じアラートが過去に発生していないかだけを調べる
- D. 原因が分かるまで、利用者への影響確認は後回しにする
**正答**: B / 測定スキル: S005 / S006 / S007
判定意図: 原因より先に現在状態・影響・正常性確認方法を見る意識を測る

### Q4(対応Lv4: 事実整理者)
**質問**: 障害対応中に、複数の確認結果と原因候補が出てきました。記録方法として適切なのはどれですか？
- A. 原因候補を中心にまとめ、確認した内容は重要なものだけ記録する
- B. 確認した事実、まだ確認していないこと、推測を分け、実施時刻とともに記録する
- C. 対応が終わった後に、覚えている範囲でまとめて記録する
- D. 自分が分かる内容であれば、他の人が見ても分かる形に整理する必要はない
**正答**: B / 測定スキル: S004 / S015 / S016
判定意図: 事実・未確認・推測・時系列を第三者が分かる形で整理できるか測る

### Q5(対応Lv5: 第一報担当者)
**質問**: アラート発生から10分経過しましたが、原因はまだ分かっていません。利用者影響の可能性があります。適切な対応はどれですか？
- A. 原因が分かるまで調査を続け、確定後にまとめて報告する
- B. 「原因調査中です」とだけメールで報告する
- C. 影響・現状・確認済み事項・未確認事項・次に行うことを整理し、緊急度に合った手段で第一報する
- D. 誤報の可能性もあるため、利用者から問い合わせが来るまで報告しない
**正答**: C / 測定スキル: S010 / S011 / S012
判定意図: 原因未特定でも、判断材料をそろえて適切な手段で第一報できるか測る

### Q6(対応Lv6: エスカレーション実行者)
**質問**: 調査を続けていますが、利用者影響があり、しばらく進展がありません。次の対応として適切なのはどれですか？
- A. もう少し調べれば分かる可能性があるため、自分だけで調査を続ける
- B. 担当者やエスカレーション先を確認し、確認済み・分かったこと・分からないことを整理して相談する
- C. 原因が分からないまま相談すると迷惑になるため、原因候補が絞れるまで待つ
- D. 状況説明はせず、詳しい人へアラート通知だけを転送する
**正答**: B / 測定スキル: S009 / S013 / S014
判定意図: 相談時機・相談先・伝達内容を整理してエスカレーションできるか測る

### Q7(対応Lv7: 類似事例活用者)
**質問**: 現在のアラートとよく似た過去事例が見つかりました。次の対応として適切なのはどれですか？
- A. 同じアラート名なので、過去事例と同じ対応をそのまま実施する
- B. 過去事例は参考にせず、最初からすべて調べ直す
- C. 対象システム・監視項目・現在状態を比較し、今回にも使える情報と、そのまま使えない情報を分ける
- D. 過去事例に復旧実績があるため、現在の影響確認を省略する
**正答**: C / 測定スキル: S008 / S002 / S006 / S020
判定意図: 過去事例を鵜呑みにせず、今回への適用可否を判断できるか測る

### Q8(対応Lv8: 初動自走者)
**質問**: 担当システムで見慣れないアラートが発生しました。初動対応の進め方として適切なのはどれですか？
- A. 原因調査を優先し、ログを確認してから影響確認や報告を行う
- B. 手順書や運用資料を参照し、対象・事象・現在状態・影響を確認して記録し、第一報と必要な相談まで順番に進める
- C. 詳しい担当者へアラートを転送し、その後の対応を任せる
- D. 過去事例と同じアラートであれば、影響確認や第一報を省略して過去の対応を実施する
**正答**: B / 測定スキル: Lv1〜Lv7の主要スキル
判定意図: 個別スキルを一連の初動プロセスとして自力で組み合わせられるか測る

### Q9(対応Lv9: 初動安定対応者)
**質問**: 普段とは異なるシステムで、見慣れない監視項目のアラートが発生しました。対応として適切なのはどれですか？
- A. 見慣れないアラートなので、詳しい担当者へすぐ任せる
- B. 普段のシステムとは異なるため、初動の確認順序もその場で考え直す
- C. 対象や監視項目に合わせて参照資料を変えながら、状態確認・影響確認・記録・第一報・相談の順序は崩さず進める
- D. 過去の対応経験がないため、原因調査から始めて状況を把握する
**正答**: C / 測定スキル: Lv1〜Lv8の主要スキル
判定意図: 対象や監視項目が変わっても、初動の基本プロセスを安定して再現できるか測る

### Q10(対応Lv10: 初動習熟者)
**質問**: 初動対応が完了しました。次回の対応をより良くするために行うこととして、最も適切なのはどれですか？
- A. 復旧したため、今回の対応は完了として記録を閉じる
- B. 原因と復旧方法だけを残し、初動で迷った点は個人の経験として覚えておく
- C. 確認漏れ・判断に迷った点・不足していた資料や連絡先を振り返り、手順書やチェックリストなどへ改善を反映する
- D. 次回も同じ担当者が対応すればよいため、資料の更新は行わない
**正答**: C / 測定スキル: S017 + Lv1〜Lv9の主要スキル
判定意図: 安定した初動対応に加え、経験を改善へ戻せるか測る
